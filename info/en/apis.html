About Arachne Interfaces

In order to guarantee the highest data interoperability, the Arachne database provides its data in several metadata formats delivered through different application interfaces.

OAI PMH

The base URI of the OAI PMH interface is http://arachne.uni-koeln.de/OAI-PMH/oai-pmh.xml. An example can be found here. Note, however, that we are currently working on a new interface for the data export.

The OAI PMH interface is specified in the Open Archives Protocol.

Books via OAI PMH

The contents of the Arachne bookbrowser can be harvested in the METS/MODS format via a separate OAI PMH interface (based on jOAI).

The base URI of this interface is http://arachne.uni-koeln.de/oai/provider. An example can be found here.

For specifications see:

http://www.loc.gov/standards/mets/

http://www.loc.gov/standards/mods/

http://dfg-viewer.de/profil-der-metadaten/

Arachne Atom Feeds

As a low level access we offer an Atom-based data interface for all of the data in Arachne. The widely accepted Atom Syndication format provides a relatively simple approach to interoperability and is easily accessible through many different sorts of clients.

Our service allows querying our full-text index and provides paged feeds of the search results. These results include the title, the persistent identifier and location, and the date of the last modification and therefore are independent of any complex metadata format.

In the near future the interface will be expanded to include several extensions described in the Open Search Standard, such as relevance scoring and geospatial and time-based query options.

The Atom Interface can be accessed from the following URL: http://arachne.uni-koeln.de/solr/atom/

Example: Basilica Aemilia

http://arachne.uni-koeln.de/solr/atom/?q=basilica+aemilia
